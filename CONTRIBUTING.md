## Contribution Guidelines

- This project is open to external contributors.
- Follow PEP 8 coding standards.
- Write clear commit messages.
- Ensure sensitive data (API keys, secrets) is never committed.
- Add documentation for any new features.

## How to Contribute

1. Fork the repository
2. Create a new branch for your feature or fix
3. Make your changes
4. Submit a Merge Request with a clear description

## Reporting Issues

- Use the provided issue templates
- Include steps to reproduce
- Mention expected vs actual behavior

## Code of Conduct

Be respectful and professional. Harassment or abusive behavior will not be tolerated.
